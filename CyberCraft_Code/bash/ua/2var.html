Змінні

Змінні утворюють фундаментальні будівельні блоки будь-якої мови програмування. Програмування скриптів shell і bash пропонує змінні, як і інші мови.
Змінна служить контейнером для зберігання даних у програмуванні. Він містить вказівник на розташування даних у пам’яті.
Оголошення змінної відбувається так:
variableName=VariableValue
variableName - це ім’я змінної, яка може містити будь-яку комбінацію букв (від a до z, від A до Z), цифр (від 0 до 9) і символів підкреслення (_).
VariableValue - це значення, яке зберігається в змінній, і воно може бути рядком чисел або логічним значенням. Символ рівності (=) використовується для присвоєння значення змінній.
Ось більш детальний приклад:
AGE = 25
В даному прикладі ми створили змінну age із значенням 25.
Після оголошення та присвоєння змінним значень ви можете отримати доступ до них за допомогою символу долара $, за яким слідує назва змінної:
AGE = 25
echo $AGE
Цей код оголошує змінну AGE зі значенням 25 і потім використовує echo для відображення значення AGE змінної. Символ $ перед назвою змінної має вирішальне значення для доступу до її значення.
Також, ми можемо змінити значення тієїж змінної:
AGE = 25
echo $AGE
AGE = 35
echo $AGE
Вивід:
25
35

Змінні тільки для читання

Ці змінні фактично робляться константами. Цей тип змінної запобігає оновлення змінної:
AGE = 25
echo $AGE
readonly AGE
AGE = 35
echo $AGE
25
line 6: AGE: is read-only
Присвоєння нового значення для змінної Age викликає помилку, а повідомлення про помилку: AGE: is read-only.

Значення unset

Ключове слово unset допомагає видалити значення з указаної змінної. Змінна залишається доступною, але друкує порожнє значення:
AGE = 25
echo $AGE
unset AGE
echo "empty":$AGE
25
empty:

Область змінних

  • Локальні змінні
  • Глобальні змінні

Глобальні змінні

Змінні, оголошені в сценарії оболонки, називаються глобальними змінними.
Доступ до глобальних змінних можна отримати в межах функції або будь-яких вкладених блоків файлу сценарію оболонки.
Змінна за замовчуванням, оголошена у файлі сценарію, називається глобальною змінною.
setAge() {
echo "Inside Function Age: $AGE"
}
AGE = 40
setAge
echo "Script Age: $tmp"

Inside Function Age: 40
Script Age: 40

Локальні змінні

Локальні змінні оголошуються всередині блоку коду або функції. Область цих змінних видима лише в межах блоку, де вони оголошені.
local variableName = VariableValue
Створимо тепер більший приклад:
setAge() {
local AGE = 25
echo "Local Variable Age: $AGE"
}
AGE = 40
setAge
echo "Global Age: $tmp"

Local Variable Age: 25
Global Age: 40

Типи даних

Сценарії Bash не є типізованою мовою, однак ви можете оголосити змінну з типом даних за допомогою команди declare. Залежно від типу змінної, вона дозволяє тип даних.
declare options variablename = value

Вивід змінних

Є дві команди: env, printenv. Обидві ці команди перелічують усі змінні середовища терміналу. Різниця між цими командами та echo полягає в тому, що ця команда призначена для відображення тексту, переданого їй в аргументах. Наприклад, виклик echo Hello, World! виведе на екран рядок "Hello, World!". Вона зазвичай використовується для виведення простих повідомлень на екран або для відображення значень змінних середовища.
Отже, основна різниця між цими командами полягає у їхній функціональності: echo виводить текст, env виводить список змінних середовища та їхні значення, а printenv лише виводить список змінних середовища без іншої функціональності.

Змінні SHELL

Це змінні, які встановлює Shell, а не користувач. Вони потрібні Shell для безперебійної роботи:
  • PWD - поточна директорія в якому відбуваютсья дії.
  • Set-Location - змінити поточну директорії на нову, де будуть відбуватися наступні дії
  • Rename-Item - перейменувати файл
  • IFS - Internal Field Separator (внутрішній розділювач полів). За замовчуванням він є пробілом, встановлений оболонкою. Використовується для розділення рядків.
  • PATH - містить розділений крапкою з комою шлях команд, налаштований на пошук команд
  • UID - друкує ідентифікаційний номер користувача
  • Home - домашній каталог поточного користувача
