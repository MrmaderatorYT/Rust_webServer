Оператор instanceof

instanceof - оператор, який потрібен, щоб перевірити, чи був об'єкт на який посилається змінна x, створена на основі будь-якого класу Y. Виходячи з цього, ми можемо зрозуміти, що цей оператор булевий і повертає значення true та false. Створімо простий приклад:

public class Main{
public static void main(String[] args){
Integer x = new Integer(1);
System.out.println(x instanceof Integer);
}
}
------------
true

Ми створили об'єкт на основі класу Integer та записали туди значення і потім ми створили перевірку: чи змінна x належить класу Integer? І так як належить, то отримали значення true. Якщо ми створили перевірку на інший клас, то отримали помилку ще до компіляції:

public class Main{
public static void main(String[] args){
Integer x = new Integer(1);
System.out.println(x instanceof String);
}
}
------------
Помилка

Також хочу зазначити, що оператор instanceof перевіряє походження об'єкта. Я раніше казав слово змінна, але надалі буду казати об'єкт (мабуть). Створімо важчий приклад:

class Animal{
}
public class Main extends Animal{
public static void main(String[] args){
Main main = new Main();
System.out.println(main instanceof Animal);
}
}
------------
true

Ми отримаємо значення true, бо об'єкт dog має всі елементи, що й клас Animal. І тоді можливо виникне питання: як ми отримаємо значення false, якщо отримаємо помилку ще до її компіляції. Створімо приклад:

public class Main extends Animal{
public static void main(String[] args){
Main main = null;
System.out.println(main instanceof Main);
}
}
------------
false

Ми отримаємо значення false, бо ми застосовуємо оператор до змінної з нульвоим значенням. Раніше я максимально пояснив про значення null. Зараз буде трішки більше інформації.
null - використовується тільки до об'єктів і тому буде просто оголошена і все. Пам'ять під неї не виділяється (пам'ять виділяється завдяки ключовому слову new.
