Посилання на метод та клас

Якщо мова йде про виконання посилання на нестатичний метод конкретного об'єкта і через оператор :: вказується назва методу. Тобто шаблон виконання посилання на нестатичний метод такий:

об'єкт :: ім'яОб'єкту;

Це було посилання на метод об'єкта, тепер перейдімо до посилання на нестатичний метод класу. В останньому випадку має значення, яи є метод статичним, чи ні.Посилання на нестатичний метод конкретного класу вказується в такому вигляді:

Клас :: ім'яМетоду;

Посилання на статичний метод...
Посилання на статичний метод формально виконується так само, як і посилання на нестатичний метод класу, однак наслідки інші. Посилання на методи можна розписати, як лямбда-виразом. Наприклад, посилання на метод об'єкта використовуючи лямбда-вираз має такий вигляд:

аргументи -> об'єкт.метод(аргументи)

Якщо рописати лямбда-виразом посилання на нестатичний метод класу, то це виглядає так:

(об'єкт, аргументи) -> об'єкт.метод(аргументи)

А посилання на статичний метод через лямбда-вираз виглядає так:

(аргументи) -> клас.метод(аргументи)

Наприклад, посилання на статичний метод:

System.out::println

Еквівалентно:

t -> System.out.println(t);

Ось приклад посилання на нестатичний метод класу виглядає так:

MyClass :: method

І це еквівалентно:

(MyClass obj, int num, char symb)->
obj.method(num, symb)

Перейдімо до останнього посилання: на конструктор. Це виглядає так:

клас::new

Як бачимо, клас написано клас написано з маленької літери, а отже це ім'я класу. Це посилання еквівалентно такому лямбда-виразу:

(аргументи) -> new клас(аргументи)
