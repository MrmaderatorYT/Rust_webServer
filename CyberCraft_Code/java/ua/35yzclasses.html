Узагальнені класи та методи

Узагальнений клас описується так просто, як і звичайний, але зрозуміло існують певні особливості. В описанні класу ми додаємо певний формальний параметр (або параметри) і вони декларуються як такий, що позначає тип даних, а у випадку створення об'єкта класу ідентифікатор типу передається, як параметр. Параметри, котрі ототожнюються в описанні узагальненого класу з типом (буду називати їх узагальненими параметрами), вказуються в кутових дужках в описанні класу після імен класу:

class Ім'я <параметр(и) типу>{
//описання
}

Узагальнені класи використовують у таких випадках, коли за допомогою класу реалізується певна "структура" або "конструкція", яка має універсальні характеристики і слабо залежить від типу даних з якими доводиться мати справу. Параметри типу просто перераховуються через кому і ці параметри використовують в описанні узагальненого класу (як ідентифікатори типу). Як параметри типу можуть використовувати в принципі будь-який ідентифікатор - головне, щоб це було не одне з зареєстрованих ключових слів (як int та інші):

class Class <X, Q>{
//описання
}

У випадку створення об'єкта на основі узагальненого класу в кутових дужках у команді створення об'єктної змінної вказують ідентифікатори типу, які слід використовувати замість узагальнених параметрів. Також ідентифікатори вказуються в кутових дужках в інструкції створення об'єкта - між назвою класу і круглими дужками з аргументами конструктора:

class Class <X, Q>{
Class <X, Q> obj = new Class <X, Q> (аргументи);
}

Також є спрощений варіант:

class Class <X, Q>{
Class <X, Q> obj = new Class < > (аргументи);
}

В данному випадку підстановка для узагальнених параметрів у випадку створення об'єкту визначається на основі ідентифікаторів указаних у перших кутових у перших кутових дужках (в інструкції оголошення об'єктної змінної).

Наслідування узагальнених класів

Щоб наслідувати універсальні класи треба так, щоб основний клас і супер клас мав однакові параметри:

class Class <X, Q>{
int a;
}
class Main <X, Q> extends Class <X, Q>{
//код
}

Також головний клас може додавати та використовувати якісь свої параметри типів:

class Class <X, Q>{
int a;
}
class Main <X, Q, Z> extends Class <X, Q>{
//код
}

Також можна мати таку ситуацію: коли клас взагалі може не бути узагальненим:

class Class <X, Q>{
int a;
}
class Main extends Class <Integer>{
//код
}

Узагальнені методи

Зазвичай узагальнені методи є статичними. Вони створюються так само, як звичайні, але мають параметри.

static <параметр> типРезультату ім'я (аргументи){
//код
}

Ось вам найпростіший вигляд:

static <X> void show(){
//код
}
