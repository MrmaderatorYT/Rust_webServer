volatile

Модифікатор volatile накладає деякі додаткові умови для читання або запису змінної. Існують деякі правила для цього модифікатора:

  • Операції читання або запису volatile-змінних є атомарими
    • Атомарна операція - операція, яка виконується повністю, або не виконується зовсім, тобто операція не може бути частково виконана та частково не виконана. Гарний приклад прислів'я "Все, або нічого"
  • Результат операції запису значень в volatile-змінну одним потоком чтає видно всім потокам програми. Виглядає це так:

class Main{
private volatile int i;
}

Якщо простими словами, то цей модифікатор каже потоку, що зміна може змінюватися та інформує потік про необхідність звертатися до останньої версії, а не до хешованої копії та своєчасно поширювати зміни.
