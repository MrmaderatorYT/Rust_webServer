Коментарі

Коментар - це скажімо нотатки в коді. Як ми знаємо, компілятор читає весь код, а коментар це проміжок в коді, який компілятор ігнорує. Всього існує три види коментарів.

  • // - коментар, який робить нечитабельним компілятору весь рядок, де цей коментар стоїть.

    int a;
    //sfdfdsfdsfdsf

  • /* */ - діапазонний коментар. Цей тип коментаря робить код нечитабельним компілятору в такому розмірі, в якому захоче користувач:

    int a;
    /* sfdfdsfdsfdsf
    2324dsdsf */

  • /** */ - коментар для документацій. Коли ви захочете читати документації, можете зустріти деякі елементи:
    • @param - використовується, щоб показати, які параметри передають в метод і яке значення вони повиннімати.
    • @return - використовується для вказівки результату, який дасть метод
    • @throw - використовується, щоб вказати, що метод викидає вийняток або помилку у випадку певного введення.
    • @since - використовується для позначення найранішої версії Java у якій був доступний цей клас або метод.
    • @author - використовується для ідентифікації автора.
    • @exception - це те саме, що й @throw, але @exception працює на більш ранішої версії.
    • {@code код } - відображає інформації як є, без html - обробки
    • @deprecated - описує застарілий клас або його член.
