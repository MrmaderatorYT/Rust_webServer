Цикл for

Цикл for не такий, як цикл while, але він є більш зручнішим у використанні. Різниця між циклом while полягає в тому, що цикл for має виконуватися певну кількість разів, коли відома кількість інтерацій. Інтерація - це повторне застосування математичної операції. Стандартний шаблок цього циклу такий:

for (ініціалізація ; умова ; математичний вираз){
команди
}

В блоці ініціалізації ми створюємо нову змінну чи записуємо існуючу.
В блоці умови ми перевіряємо умову, яку ми реалізовуємо в цьому блоці. Тут у нас буде синтаксис написання блоку if, де будемо записувати умову, яка перевіряється.
В блоці математичного виразу ми записуємо... математичний вираз. Зазвичай це інкремент або декремент.
Ось простий вигляд циклу for.

for(int k = 1; k < 5){
System.out.println(k);
}

Ми отримаємо в консолі чотири значення: 1,2,3,4. Як ми бачимо, що спочатку виконується перший блок, тобто оголошення змінної k. Другий блок перевіряє умову, якщо отримаємо значення true, тобто в нашому випадку ми отримаємо значення true доки змінна k буде менше за 5, то виконується команда, а саме код, який знаходиться у фігурних дужках. Після цього виконується третій блок, тобто математичний вираз, в нашому випадку k++. Якщо ми отримаємо значення false в другому блоці, то цикл завершує виконання. Ось ще один приклад:

int a;
for(i=1; i<5;i++){
System.out.println(i);
}

Різниці з попереднім циклом практично немає, але в даному випадку, ми можемо використовувати змінну по всьому класі, а в першому варіанті ми змінну k можемо використовувати тільки в тілі циклу. Перейдімо до 3 прикладу:

int n, s;
for(n=1, s = 0; i>=s;s++, n++){
System.out.println(n, s);
}

Тут ми бачимо оголошення змінних n, s. В циклі присвоїли значення цим змінним. В другому блоці ми перевіряємо дані, тобто створили умову, а в третьому блоці ми виконуємо команди, якщо при перевірці в другому блоці ми отримаємо істину (true). Настав час для 4 прикладу:

int s = 0, k = 1, n = 10;
for(; k<=n ;){
s+=k*k;
k++;
System.out.println(s, k, n);
}

Тут ми не оголошували змінну, бо вона вже створена й присвоєна до якогось числа. В другому блоці ми створили перевірку, а в третьому блоці ми не написали математичний вираз, бо ми його написали в блоці виконання коду. Якщо другий блок виведе істину, тобто значення true. Перейдімо до останнього прикладу, коли цикл повністю пустий:

int s = 0, k = 1, n = 10;
for(; ;){
s++;
k++;
n++;
System.out.println(s, k, n);
}

Це нескінченний цикл, тобто умова завжди буде true.
