Умовний оператор

Умовний опертор або структура if-else

Умовний оператор дозволяє виконувати різні блоки команд відповідно від істониї або хибної певної умови.

Для оголошення умовного оператора, потрібно написати ключове слово if Ось вам стандартний вигляд умовного оператору:

if (умова){
код
}

Створімо приклад:

import java.util.Scanner;
public class Main{
public static void main(String[] args){
Scanner scanner = new Scanner(System.in);
int a = scanner.nextInt();
if (a>100){
System.out.println("Число більше 100");
}
}
}

Це спрощений вигляд умовного оператора. Якщо ми введемо число більше 100 т отримаємо в консолі: Число більше 100, але якщо ми введемо число менше або дорівнює 100, ми нічого не отримаємо знову ж таки із-за спрощеної структури.
Повна умова має ключове слово else
Повна структура умовного оператору виглядає так:


if (умова){
код
}
else {
код
}

Отже, логіка виконання умовного оператора така: якщо умова в блоці if правдива, виконується код в блоці if, якщо умова в дужках не істина, тобто хибна - виконується блок else в цьому блоці ми не ставимо умови, бо він буде виконуватися коли не виконається блок if.
Створімо приклад:

int a=5;
if (a<4){
System.out.println("no")
}
else {
System.out.println("yes")
}

Тут ми створили числову змінну а і присвоїли їй число 5. На наступному рядку створили перевірку: чи змінна а менше 4. Якщо так, то отримаємо в консолі текст no, якщо ні(більше або дорівнює 4) отримаємо yes.


Перейдемо до іншого типу операторів - вкладених. Вкладений тип має додаткове ключове слово else if

if (умова){
код
}
else if(умова){
код
}
else {
код
}

else if записується після блоку if. Блоків else if може бути безліч, хоч 100. Логіка виконання так: якщо умова в блоці if хибна, виконується блок else if, якщо в цьому блоці виконується хибна умова, виконується інший блок і так далі, якщо існує блок else, то після всіх хибних умов else if виконується код в блоці else, якщо блока else нема, то нічого не отримаємо.

int a = 11;
if (a<11){
System.out.println("first");
}
else if(a==10){
System.out.println("second");
}
else if(a==12) {
System.out.println("third");
}

Тут ми створили змінну та присвоїли число 11. Потім створили першу перевірку, якщо змінна менше 11, то в консолі отримаємо текст first, якщо ми отримаємо хибне значення, тобто якщо змінна a більше або дорівнює 11. Наступний блок - якщо змінна дорівнює 10, а третій, якщо 12. В цьому коді ми Нічого не отримаємо, бо перевірка трьох блоків не входить в діапазон числа 11, а блока else нема

int a = 0;
if (a==0){
System.out.println("first");
}
else if(a>=0){
System.out.println("second");
}
else{
System.out.println("third");
}

Як бачимо у нас дві перевірки. Ми отримаємо в консоль first і все, тобто тільки його, а не first та second так як у нас змінна входить ще й в другу перевірку. Ми не отримаємо значення, бо після отримання значення true тобто істини, перевірка завершується

.

Є ще інший приклад:

int a = 0;
if (a==0){
System.out.println("first");
}
if(a>=0){
System.out.println("second");
}
else{
System.out.println("third");
}

Тут ми отримаємо в консолі два значення, а саме first та second, бо тут два блоки if. Перший блок - це спрощений вигляд, а другий блок if - повний, тобто else прикріплюється до останнього блоку if.
