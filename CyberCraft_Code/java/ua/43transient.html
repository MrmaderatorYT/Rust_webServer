Модифікатор transient

Цей модифікатор використовується в процесі серіалізації та десеріалізації об'єктів. Припустімо в нас є об'єкт, який має змінні, а ці змінні - значення. Усе це назвивається станом об'єктом.
Серіалізація - конвертація стану об'єкта в послідовність байтів. Ці байти зберігаються зазвичай у якомусь файлі.
Десеріалізація - зворотній процес серіалізації.
transient корисне в кількох сценаріях:
1. При використанні його для похідних змінних.
2. Це корисно для змінних, які не представляють стан об'єкта
3. Для будь-яких несеріалізованих посилань.
Щоб здійснити серіалізацію потрібно реалізувати інтерфейс Serializable:

class Class implements Serializable{
//код
}

Призначення цього модифікатора є в тому, коли виникає необхідність зберегти об'єкт класу. Модифікатор transient встановлюється перед оголошенням внутрішньої змінної класу, значення якої не повинно зберігатись у випадку збереження об'єкту. Завдяки цьому модифікатору можна вказати дані, які не потрібно зберігати при серіалізації об'єкту:

class Book{
String name;
String author;
int year;
transient String id;
}

В цьому випадку змінна id не зберігається для запису в файл
