Інкапсуляція. Класи

Cтворімо клас Children:

public class Main{
public static void main(String[] args){
}
}
class Children{
int age;
String name;
void print_info(){
System.out.println("Ім'я" + name);
}
}

Тут ми створили метод, але метод не публічний, тобто його не можна використовувати в інших класах.
Конструктори - альтернатива звичайних методів. Це такі спеціальні методи. Вони викликаються при створенні нового об'єкта даного класу. Конструктори виконують ініціалізацію об'єкта та носять ім'я класу. У нас є клас Children та видалимо метод та запишемо конструктор:

public class Main{
public static void main(String[] args){
}
}
class Children{
int age;
String name;
public Children(String name, int age){
this.name = name;
this.age = age;
}
}

this - ключове слово, яке вказує посилання на поточний екземпляр класу. Через це ключове слово ми можемо звертатися до змінних, мотодів об'єкта, а також викликати його конструктори. Після ключового слова ми ставимо крапку та звертаємося до змінної, яку створили і присвоїли до тією, яку ми створили в конструкторі. Перейдімо до головного класу Main. Звернімося до класу Children. Якщо чесно, то ми створюємо об'єкт на основі класу Children, а отже запишемо:

public class Main{
public static void main(String[] args){
Children children = new Children(10, "Ann");
}
}
class Children{
int age;
String name;
public Children(String name, int age){
this.name = name;
this.age = age;
}
}

Але в дужках потрібно записати дані, а саме вік, тобто age, та ім'я - name. Треба записувати в тому порядку, як ми записували в методі. Потім вивиедемо в консоль:

public class Main{
public static void main(String[] args){
Children children = new Children(10, "Ann");
System.out.println(children.age);
System.out.println(children.name);
}
}
class Children{
int age;
String name;
public Children(String name, int age){
this.name = name;
this.age = age;
}
}

Перейдімо до інкапсуляції.
Інкапсуляція - приховування даних у середині деякої області видимості. Для цього використовується методи доступу. Вони називаються Геттери та Сетери(Getters and Setters). Ми створили в класі Children змінну int. Якщо ми перед цією змінною напишемо модифікатор доступу private, то ми не зможемо виводити змінну в класі Main. Та й взагалі... Ми не зможемо використовувати її в інших класах. Для цього й існує Getter та Setter. Ми перезапишемо клас Children:

public class Main{
public static void main(String[] args){
Children children = new Children("Ann");
children.setAge(10);
System.out.println(children.getAge);
System.out.println(children.name);
}
}
class Children{
public int getAge() {
return age;
}
public void setAge(int age) {
this.age = age;
}
public String getName() {
return name;
}
public void setName(String name) {
this.name = name;
}
private int age;
String name;
public Children(String name){
this.name = name;
}
}
--------------------
10
Ann

Це ми створили геттер та сеттер в класі Children. Ми створили для змінній age та name. Навіщо ми оголосили змінну private до змінної та використовувати конструктори, якщо можна лише оголосити змінну public до всіх?
Просто бува таке... Навіть дуже часто, що в процесі виконання програми змінній буде передано некоректне значення. Наприклад для значення віку буде передано від'ємне значення, що не повинне бути.
