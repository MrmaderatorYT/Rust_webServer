Абстрактні класи, методи

Абстрактні класи та методи - це як ідеї для створення речей, але без деталей. Подумамо про них як про плани, які ми можеш використати для будівництва будинку, але без докладного намалювання кожного кімнати.

Абстрактні методи

Абстрактні методи - методи, в яких існує можливість обмежитися оголошенням методу, а саме вказати його сиґнатуру, але не описувати тіло методу. Кожен абстрактний метод описується ключовим словом abstract. Абстрактні методи взагалі не можа описати, а тільки оголосити:

public abstract void one();

Абстрактні класи

Абстрактний клас - клас, який містить хоча б один абстрактний метод. Навіть, якщо він клас оголосити завдяки ключовому слову abstract, то він також буде абстрактним. На основі абстрактних класів неможливо створити об'єкт із-за чого виникаєє логічне питання: навіщо вони взагалі? Зазвичай абстрактні класи використовують для успадкування. Простими словами, на основі абстрактного класу створюються шляхом успадкування підкласи, в яких описуються з програмним кодом абстрактні методи з суперкласу. Також, якщо в підкласі, який створюється на основі абстрактного суперкласу, деякі абстрактні методи залишити неописаними, то такий клас вважається абстрактним:

public abstract class Main{
abstract void a();
}

Також повернімося до створення об'єктів. Хоча на основі абстрактного класу неможливо створити об'єктну змінну, проте можна створити об'єктну змінну абстрактного класу. Така змінна може посилатися на об'єкт підкласу, створеного на основі даного абстрактного класу. Створімо абстрактний клас:

public abstract class Main{
private int age;
public int getAge(){
return age;
}
}

Тут ми створили абстрактний клас, записали змінну age. Також ми створили базовий функціонал класу. Завдяки цьому, ми можемо наслідувати цуй абстрактниц клас для опису функціоналу. Як вище було сказано, ми не можемо використовувати конструктор абстрактного класу для створення об'єкта на його основі, тобто так:

Main main = new Main(); // неправильно

Ось вам приклад абстрактних класів та методів:

//абстрактний клас фігури
abstract class Figure{
float x;
float y;
Figure(float x, float y){
this.x = x;
this.y = y;
}
//абстрактний метод для отримання периметра
public abstract float getPerimeter();
//абстрактний метод для отримання площі
public abstract float getArea();
} //похідний клас прямокутника
class Rectangle extends Figure{
private float width;
private float height;
//конструктор, який звертається до конструктора класу Figure
Rectangle(float x, float y, float width, float height){
super (x,y);
this.width = width;
this.height = height;
}
public float getPerimeter(){
return width * 2 + height * 2;
}
public float getArea(){
return width*height; }
}

Отже, навіщо нам це потрібно?

Шаблони для створення нових речей: Абстрактний клас може бути як шаблоном для створення нових класів. Наприклад, у нас є абстрактний клас "Транспортний засіб". Можемо створити підкласи, такі як "Автомобіль" чи "Велосипед", які успадковують основні риси від "Транспортного засобу".

Змушує наслідування: Коли ми робимо метод абстрактним, ми кажемо: "Хей, якщо ти хочеш бути класом, який використовує цей абстрактний клас, тобі треба мати цей метод!" Це допомагає нам забезпечити, що всі підкласи будуть мати однаковий набір функцій, але можуть реалізувати їх по-різному.

Вибіркова реалізація: Абстрактні методи дозволяють нам вказати, що ми хочемо, але не як саме ми хочемо це зробити. Наприклад, у нас є абстрактний метод "Підрахунок площі". Ми можемо вказати, що кожен клас, який успадковує цей абстрактний клас, повинен мати метод для підрахунку площі, але ми не кажемо, як саме це робити. Так кожен підклас може реалізувати цей метод по-своєму, в залежності від його конкретних потреб.

Отже, користуючись абстрактними класами та методами, ми створюємо загальні шаблони, які можна використовувати для створення специфічних реалізацій, забезпечуючи при цьому структурованість і гнучкість програми.
