Значення null

null - це значення, а не тип, тобто наприклад в цьому коді, змінна a має значення 5:

int a = 5;

Так само, які 5, null має значення. Також null використовується тільки до об'єктів, тобто до примітивів, як int і так далі, ми не зможемо використовувати значення null:

char a = null; //помилка

Тобто це значення використовується таким способом:

String a = null;

Також, як кожен примітив має значення за замовчуванням, наприклад у int - число 0, а у boolean - false, то значення за замовчуванням усіх об'єктів є null. Також, треба зазначити, що null це не є об'єктом та не є типом знову. Це просто спеціальне значення для об'єктів. Ми, навіть, можемо використати це значення до будь-якого типу, наприклад:

String a = null;
Integer b = null;
String a1 = (String)null;
Integer b1 = (Integer)null;

Як я казав, то null не може використовується до примітивних типів, бо значення null не може бути переведеним до того значення, яке ми вказали в типі, тобто:

double a = null; //помилка

Але кожний примітив має клас оболонку. І якщо створити об'єкт на основі класу оболонки, а потім присвоїти до примітивного типу, ми отримаємо помилку компіляції NullPointerException:

Integer a = null;
int b = (int)a;

Це все із-за автоупаковки.

Автоупаковка, розпакування

Автоупаковка - перетворення примітивних типів на еквівалентні об'єкти, реалізована автоматично.
Розпакування - зворотній процес автоупаковки.

Тобто все це можна вказати як: процес перетворення об'єктів у відповідні їм примітивні типи. Приклад коду для автоупаковки та розпакування представлений нижче:

Integer integer = 9; // автоупаковка

int in = 0;
in = new Integer(9); // розпаковка

Коли використовується автоупаковка та розпакування?

Автоупаковка застосовується компілятором Java за таких умов:

  • Коли значення примітивного типу передається в метод як параметр методу, який очікує на об'єкт відповідного класу-оболонки.
  • Коли значення примітивного типу надається змінної, відповідного класу оболонки.

Повернімося до оператору instanceof. Він буде виводити значення false в усіх випадках, якщо об'єктна змінна має значення null. Також ми можеемо порівнювати значення null використовуючи оператори == та !=, але не до арифметичних та логічних операторів.
Якщо порівнювати два значення null між собою, то отримаємо значення true.
