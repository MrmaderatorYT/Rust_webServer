Обробка помилкових ситуацій (try-catch)

Система обробки помилкових базується на блоці try-catch. Цей блок допоможе обробити певну ділянку коду, що коли вона з'явиться, то не призведе до завершення програми, а просто виконається код. Ось вам стандартний шаблон:

try{
код, який може призвести до помилки
}
catch(класПомилки змінна){
опрацювання помилки
}

Отже тепер роз'яснення. В блоці try ми записуємо код, який може в якийсь момент спрацюватии не правильно.
В блоці catch ми вимоловлюємо цю помилку, вказавши який клас помилки буде ловити, після чого ви описуєте як її оброблювати.
Клас помилки - це те що в дужках catch
Ось приклад:

import java.util.Scanner;
public class Main{
public static void main(String[] args){
Scanner scanner = new Scanner(System.in);
try{
int x = scanner.nextInt();
}
catch(InputMismatchException e){
System.out.println("Не є числом");
}

Отже ми створили змінну x, завдяки якої будемо вводити значення типу int в консоль. Але буває момент, коли ми введемо не число і тоді в запуститься код, який в блоці catch. В нашому випадку текст: Не є числом. Ось вам деяки класи помилки

  • ArithmeticException - помилка пов'язана з виконанням математичних операцій. Для простого прикладу ми не можемо ділити числа на нуль.
  • ArrayStoreException - помилка пов'язана зі спробою записати в масив об'єкт типу який не відповідає типу елементів масиву. Для прикладу, ми створили масив типу String та записали туди елемент типу char.
  • ClassCastException - помилка пов'язана зі спробою некоретного приведення типів.
  • IllegalArgumentException - ця помилка пов'язана з некоретною передачею аргументів методу.
  • NumberFormatException - помилка, котра виникає за спроби перетвоити на число некоректне текстове представлення для числа
  • IndexOutOfBoundsException - помилка, пов'язана, наприклад, з некоретно вказаним індексом масива
  • StringIndexOutOfBoundsException - відповідає помилці, яка виникає з некоректно вказаним індексом елемента в текстовому рядку.
  • NegativeArraySizeException - помилка пов'язана зі спробою створити масив від'ємного розміру
  • NoSuchElementException - помилка пов'язана зі зверненням до неіснуючого елемента
  • NullPointerException - помилка пов'язана з використанням пустого посилання (значення null) замість посилання на об'єкт
    • null використовується для представлення навмисної відсутності значень.
  • UnsupportedOperationException - помилка пов'язана зі спробою виконати операцію, яка не підтримується.
  • ArrayIndexOutOfBoundsException - помилка виникає під час звернення до елемента за індексом, який виходить за присутній діапазон значень.
  • ClassNotFoundException - помилка пов'язана з неможливістю отримання доступу до класу.
  • IllegalAccessException - помилка виникає під час отримання доступу до ресурсу.
  • InstantionException - помилка пов'язана з неможливістю створення об'єкта (наприклад, на основі абстрактного класу)
  • InterruptedException - помилка, пов'язана з перериванням одного потоку іншим потоком.
  • NoSuchFieldException - помилка, пов'язана з відсутністю поля.
  • NoSuchMethodException - помилка пов'язана з відсутністю метода.

Також є опрацювати декілька помилок:

import java.util.Scanner;
public class Main{
public static void main(String[] args){
Scanner scanner = new Scanner(System.in);
try{
int x = scanner.nextInt();
}
catch(InputMismatchException e){
System.out.println("Не є числом");
}
catch(NumberFormatException e){
System.out.println("Не той формат чисел");
}
