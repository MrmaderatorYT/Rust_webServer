Інтерфейси

Інтерфейс - урізана копія абстрактного класу. Інтерфейс містить оголошення методів або/і статичних констрант. В інтерфейсі методи тільки оголошуються. Процес реалізації інтерфейсу в класі нагадує успадковування абстрактного суперкласу в підкласі. У випадку з інтерфейсами, клас, який реалізує інтерфейс, має містити описання всіх методів, оголошених в інтерфейсі. Важлива обставина пов'язана з тим, що один і той же клас може реалізувати одночасно декілька інтерфейсів. Реалізація інтерфейсів описується так само, як і звичайний клас, але замість ключового слова class, ми повинні записати interface.
Навіщо взагалі інтерфейси?
Справа в тому, що в Java заборонено багатократне спадкування: у підкласу може бути один і тільки один суперклас. Це серйозне обмеження, бо, наприклад в мові С++, клас може успадковувати одночасно декілька класів.
Ще один ключовий момент: всі методи, які оголошені в інтерфейсі є публічними. Тобто модифікатор доступу public не потрібен.

interface MyInterface{
int number = 100;
}

Створимо простий приклад реалізації інтерфейсів:

interface A{
int number = 50;
int getNumber(double x);
char getSymbol(int y);
}
class MyClass implements A{
public int getNumber(double x){
return (int) x;
}
public char getSymbol (int n){
return (char)('A' + n);
}
}
class Main{
public static void main(String[] args){
MyClass obj = new MyClass();
System.out.println("Статична константа: " + MyClass.number);
System.out.println("Ціле число: "+obj.getNumber(20.5));
System.out.println("Символ: "+obj.getSymbol(3));
}
}
Статична константа: 50
Ціле число: 20
Символ: D

return(int) x - приведення типів: змінна x типу double, а ми замінили на int
return (char)('A' + n); - ми перевели порядковий номер букви в текст.
