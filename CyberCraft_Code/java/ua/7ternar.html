Тернарний оператор

Тернарний оператор - заміна if-else блоку. Тобто цей оператор виконує роботу блоку if та блоку else, тобто блок else if, тернарний оператор не сприймає

Ось вам стандартний вигляд умовного тернарного оператора:

вираз_№1 ? вираз_№2 : вираз_№3;

Тепер розберімося

Вираз №1 - позначає будь-який вираз, обчислення якого дає логічне значення (тип boolean), але для початку створімо змінну a, а в виразі ми створимо іншу змінну:

int a = 10;
int b = ? вираз_№2 : вираз_№3;

Ми створили змінну b типу int. Після дорівнює ми створюємо перевірку на вираз, це те саме, що умова в дужках. Зазвичай позначається умова в дужках, але не обов'язкого:

int a = 10;
int b = (a>5) ? вираз_№2 : вираз_№3;

В виразі вираз №2 та вираз №3 - виконує код.
Вираз №2 - це той код, який виконується при істиній істиній умові, тобто код, який знаходиться в блоці if.
Вираз №3 - це той код, який знаходиться в блоці else, тобто код який виконається в тому випадку, якщо вираз №2 отримає хибність.

Отже ось повний код з використанням тернарного оператором:

int a = 10;
int b = (a>5) ? ++a : --a;

Тут ми ставимо крапку з комою в кінці виразу.
Є один нюанс при використанні тернарного оператора: цей оператор працює тільки з присвоєнням нової змінної того ж типу чи тієї ж змінної:

int a = 10;
a = a>5 ? ++a : --a;

Та між:

int a = 10;
int b = (a>) ? ++a : --a;

Різниці не буде.
Також є ще можливе питання: що працює швидше? Тернарний оператор чи звичайний умовний оператор.... Швидкість виконання однакова.
