Перерахування (enum)

Enum - це спеціальний тип даних, який дозволяє змінній бути набором попередньо визначених констант. Змінна має дорівнювати одному із попередньо визначених для неї значень. Типовим прикладом є напрямки горизонту: значення північ, південь, захід, схід. Воне є константами, тож можна, хоч і не обов'язково записати з великої літери. Ось приклад:

public enum Compass{
NORTH, SOUTH, WEST, EAST;
}

Також enum повинен мати модифікатор доступу. Якщо enum не є вкладеним в клас, то він має бути відкритим. Якщо enum буде створений як в окремому файлі, то модифікатор доступу private не можна використовувати. Але, якщо enum є вкладеним у клас, він може бути приватним:

public class Main{
private enum Data{
NORTH, SOUTH, WEST, EAST;
}
}

Щоб отримати дані з enum, можна використовувати цей код:

public class Test{
public static void main(String[] args){
System.out.println(Compass.NORTH);
System.out.println(Compass.SOUTH);
System.out.println(Compass.WEST);
System.out.println(Compass.EAST);
}
}
-----------------------
NORTH
SOUTH
WEST
EAST

Також, можна отримати значення використовуючи for-each цикл та методу .values():

for(Main s : Main.values()){
System.out.println(s);
}
Також дуже зручно використовувати оператор вибору:
Main var = Main.EAST;
switch(var){
case NORTH:
System.out.println("Північ");
break;
case SOUTH:
System.out.println("Південь");
break;
case WEST:
System.out.println("Захід");
break;
case EAST:
System.out.println("Схід");
break;
}

В нашому випадку, ми отримаємо в консолі "Північ", бо в нас всі відповіді для зміної var є істині, якщо прибрати ключове слово break, то ми отримаємо всі варіанти відповіді.
Але навіщо нам використовувати enum, якщо можна створити свої змінні та все? Перш за все, як на початку було сказано, перерахування - набір констант enum розглядаються, як типи даних, і знову ж таки, завдяки константам. Одне із ключових урізаних можлвостей enum - неможливість успадковувати.
