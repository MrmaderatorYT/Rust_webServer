Статичні поля та методи

Основна ідея статичних членів класу полягає в тому, що статичний член класу є "загальним" для всіх об'єктів цього класу та існує незалежно від наявності або відсутності в програмі об'єктів класу. Тому звертатися до статичного члена можна не тільки через об'єкт класу але і безпосередньо через клас. Причому останній спосіб є значно кращим.
Для цього ми використовуватимемо ключове слово static
Створимо приклад без статичного елемента. Створимо новий клас в новому файлі назвою Cat:

public class Cat{
int type;
}

І тепер в головному методі main ми запишемо код:

public class Main{
public static void main(String[] args){
Cat orangeCat = new Cat();
orangeCat.type = 1;
Cat blackCat = new Cat();
blackCat.type = 3;
System.out.println("Orange: " + orangeCat.type);
System.out.println("Black: " + blackCat.type);
}
}
/*Orange: 1
Black: 3*/

А тепер, ми змінну type оголосимо статичною:

public class Main{
public static void main(String[] args){
Cat orangeCat = new Cat();
orangeCat.type = 1;
Cat blackCat = new Cat();
blackCat.type = 3;
System.out.println("Orange: " + orangeCat.type);
System.out.println("Black: " + blackCat.type);
}
}
/* Orange: 3
Black: 3 */

Ми отримали 3, а не 1, бо змінна в нас одна на всіх. І кожного разу ми замінюємо її.

Статичні класи

public class One{
public static class Two{
public int type;
}
}
Це ми створили клас, як новий файл. Перйдімо в клас Main та оголосимо змінну:
public class Main{
public static void main(String[] args){
One.Two two = new One.Two();
two.type = 50;
System.out.println("Text: " + two.type);
}
}
Text 50

Для використання статичних методів чи змінних, чи класів, нам не потрібно створювати об'єкт даного класу. Також існують даякі головні моменти використання модифікатора static

  • Ми не можемо отримати доступ до не статичним членам класу, в середині статичного метода чи блока. Ось приклад, який призведе до помилки:
    • public class Counter{
    private int count;
    public static void main(String[] args){
    System.out.println(count); //помилка
    }
    }
  • Ми не можемо перевизначити (Override) статичні методи.
