Cursuri anonime

Destul de des este nevoie de a crea o clasă care este folosită o singură dată. În astfel de cazuri, sunt folosite clase anonime
Clasuri anonime - clase care nu au „nume propriu”. Există două moduri de a crea clase anonime: prin moștenirea unei clase abstracte și prin implementarea unei interfețe. Să presupunem că avem o clasă abstractă care are numele condiționat „SuperClass”. Vom crea o clasă anonimă prin moștenirea SuperClass. Dar există o caracteristică a claselor anonime - un obiect este creat pe baza acestei clase. Cu alte cuvinte, nu are sens să vorbim despre descrierea unei clase anonime, dacă nu este vorba despre crearea unui obiect. În principiu, un obiect poate fi și anonim, dar poate fi și cazul în care o referință la un obiect creat pe baza unei clase anonime să fie scrisă într-o variabilă obiect. O referință la un obiect dintr-o clasă anonimă poate fi scrisă într-o modificare SuperClass și obținem următorul șablon:

SuperClass var = new SuperClass(argumente){
//descriere
};

Adică însăși construcția creării unui obiect bazat pe o clasă anonimă începe așa cum ar trebui cu construcția new, după care constructorul superclasei este specificat în curly paranteze - codul care conține descrierea metodelor abstracte din SuperClass. Dacă vorbim despre un obiect anonim creat pe baza unei clase anonime, atunci cel mai probabil, un astfel de obiect anonim este transmis ca argument metodei sau metoda este apelată de la aceasta. În orice caz, se folosește o construcție de forma:

new SuperClass (argumentes){
//descrierea metodelor
}

Acum să creăm un exemplu pentru crearea unei clase anonime prin inițializarea interfeței. Abordarea aici este exact aceeași ca și în cazul moștenirii clasei - cu excepția că avem de-a face cu o interfață:

Variabilă de interfață = new Interface(){
//descrierea metodelor
};

Un obiect anonim al unei clase anonime, cu condiția să implementeze interfața:

nouă Interfață(){
//descrierea metodelor
}
