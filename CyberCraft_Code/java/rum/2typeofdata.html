Tipuri de date

Tipuri de date - capacitatea de a accepta valorile specificate pentru variabilă. În Java, există două tipuri de date - primitive și pointer.

Există un total de 8 tipuri de date

  1. byte - tip care acceptă numai numere întregi. Intervalul de numere este de la -128 la 127 (numărul 0 este și aici) consumă opt biți de memorie.

    2. byte a = 120;

  2. short- poate accepta, de asemenea, numai numere întregi, dar intervalul este mult mai mare: de la -32768 la 32767, dar deoarece intervalul este mai mare, utilizarea memoriei va fi și ea mai mare, și anume 16 biți

    3. short b = 20;

  3. int - acceptă numai numere întregi în intervalul de la -2 147 483 648 la 2 147 483 647 și ocupă 32 de biți de memorie

    4. int a = 203452;

  4. long- care sunt cele anteriore, acceptă numere întregi și ocupă 64 de biți de memorie. Acceptă intervalul de la -9223372036854775808 la 9223372036854775807
    ! Dacă introduceți valori mai mari decât intervalul int, adică, aproximativ, peste 3 milioane vor fi o eroare. Eroarea este că compilatorul percepe valoarea long de int.Pentru a remedia eroarea, trebuie doar să adăugați o literă majusculă L
    L - abrevierea lui long. Pentru a remedia eroarea, trebuie doar să adăugați o literă majusculă int are o clasă wrapper de Integer.

    5. long a = -15023125;

  5. float - acceptă numai numere din intervalul de la 3,4*10^-38 la 3,4*10^38 acest tip poate accepta atât numere întregi, cât și numere fracționale. Ocupă 32 de biți de memorie.
    Care long, floattrebuie să adăugați o clasă wrapper. ÎNfloat - este... Același flotor, dar cu majusculă, adică F

    6. float a = -1.5;

  6. double - acceptă numai numere din intervalul 1,7*10^308 modulo și ocupă 64 de biți de memorie. Diferență între double și float nu este doar în gamă, ci și în precizie. Exemplu, float poate obține valoarea: 1,6893, double1.6894, pentru că am rotunjit până la zece miimi și așa cum doubleacceptă mai multe valori după virgulă decât float, atunci datele vor fi mai precise.

    7. double a = -15023125.225484;

  7. char - tip de caracter. Poate percepe litere. Dimensiunea memoriei este de 16 biți.
    char este indicat între ghilimele, dar ghilimelele sunt simple, adică aici: ' 'poti pune mai mult de un caracter in aceste ghilimele, este ca un spatiu obisnuit, numar, litera. Pe scurt, orice caracter din tabelul Unicode, cunoscut și sub numele de utf-8.

    8. char a = 'q';

  8. String Imediat spun că acest tip este scris cu majuscule, în al doilea rând, textul este scris între ghilimele " ". De asemenea, multe caractere pot fi scrise în acest tip, adică, spre deosebire de char, putem scrie cel puțin „Eneida” lui Kotlyarevsky aici.

    9. String a = "Glory to Ukraine";

    Majoritatea compilatoarelor nu colorează String deoarece, așa cum am spus, nu este un tip de date. În cod, nici nu voi colora String

  9. boolean - tip logic. Acceptă doar două valori, și anume true și false

    10. boolean a = true;

Am spus că sunt 8 tipuri de date în total, dar aici sunt 9... Gluma este că String nu este un tip de date primitiv. Acesta este un tip de indicator, adică un obiect. Primitivele ies în evidență într-o altă culoare decât alb-negru obișnuit. Să mergem la tipuri de difuzare


Type casting


Aici vom folosi operandul
Operand - argumentul operației. Castingul de tip este împărțit în două tipuri: explicit și implicit (sau automat). Conversia de tip explicit și implicit se realizează automat după o anumită logică: dacă expresia numerică conține un operand de tipul double, atunci toți ceilalți operanzi, rezultatul expresiei este automat extins la tipul double

double a2 = 0.02;
