Comentarii

Comentariul este, să zicem, note din cod. După cum știm, compilatorul citește întregul cod, iar un comentariu este un gol în cod pe care compilatorul îl ignoră. Există trei tipuri de comentarii.

  • // - un comentariu care face ca întreaga linie în care comentariul pare să nu fie citită de compilator.

    int a;
    //sfdfdsfdsfdsf

  • /* */ - comentariu de gamă. Acest tip de comentariu face codul imposibil de citit de către compilator în dimensiunea dorită de utilizator:

    int a;
    /* sfdfdsfdsfdsf
    2324dsdsf */

  • /** */ - comentariu pentru documentare. Când doriți să citiți documentația, puteți întâlni câteva elemente:
    • @param - este folosit pentru a arăta ce parametri trec metodei și ce valoare ar trebui să aibă aceștia.
    • @return - este folosit pentru a indica rezultatul pe care îl va produce metoda
    • @throw - este folosit pentru a indica faptul că o metodă va arunca o excepție sau o eroare dacă este dată o anumită intrare.
    • @since - este folosit pentru a indica cea mai veche versiune de Java în care această clasă sau metodă a fost disponibilă.
    • @author - folosit pentru a identifica autorul.
    • @exception - це те саме, що й @throw, dar @exception funcționează pe o versiune anterioară.
    • {@code cod } - afișează informațiile așa cum sunt, fără procesare html
    • @deprecated - descrie o clasă sau un membru depreciat.
