volatil

Modificatorul volatile impune unele condiții suplimentare pentru citirea sau scrierea unei variabile. Există câteva reguli pentru acest modificator:

  • Operațiile de citire sau scriere a variabilelor volatile sunt atomice
    • Operațiune atomică - o operație care este efectuată complet, sau nu este efectuată deloc, adică operația nu poate fi efectuată parțial sau parțial neefectuat. Un bun exemplu al proverbului „Totul sau nimic”
  • Rezultatul operațiunii de scriere a valorilor în variabila volatile citită de un fir este vizibil pentru toate firele de execuție ale programului. Arată astfel:

class Main{
private volatile int i;
}

În termeni simpli, acest modificator spune firului de execuție că modificarea este supusă modificării și informează firul de execuție să facă referire la cea mai recentă versiune, nu la copia codificată, și să propage modificările în timp util.
