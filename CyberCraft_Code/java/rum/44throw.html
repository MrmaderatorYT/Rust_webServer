Оператор throw

Вийняток, доки воно не викинуто в систему, або коли воно вже перехоплене блоком catch - є об'єктом класу Exception (якщо точніше Throwable). Вся логіка роботи вийнятуів - особлива логіка поведінки JVM, як у її середовищі виконання викинуто вийняток. Ми можемо завжди вкинути помилку в JVM, яку щойно перехопили. І для цього існує ключове слово throw:

throw вийняток;

вийняток - змінна, яка є в юлоці catch:

try{
int d = 210;
}
catch(Exception except){
System.out.println("перехопили вийняток");
throw except;
}

У цьому коді ми перехопили вийняток, вивели на екран напис про це і викинули його знову. Повторно викинути вийняток не може бути захоплений іншими блоками catch цього ж try-блоку. Також можна створити об'єкт виключення. Це просто об'єкт типу Exception або успадкованого від нього класу. І викинути його:

try{
throw new RuntimeException();
}
catch(Exception except){
System.out.println(Перехопили вийняток");
throw except;
}

У цьому прикладі ми створили новий об'єкт - виключення RuntimeException і відразу викинули його за допомогою оператора throw. Воно буде відразу захоплено блоком catch, так як тип RuntimeException успадкований від типу Exception. Блок catch захоплює об'єкти-виключення всіх класів, успадкованих від класу Exception
