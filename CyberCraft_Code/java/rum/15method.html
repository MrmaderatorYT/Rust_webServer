Metode

O

Metodă este un bloc de cod care este executat numai atunci când este apelat. De asemenea, un alt nume pentru metode este funcții.

De ce metode?

Sunt necesare metode pentru reutilizarea codului: scrieți un cod într-o metodă și apelați-l când aveți nevoie, în loc să îl rescrieți. Iată un exemplu:

public class Main{
public static void main(String[] args){
cod
}
public static void method(){
cod
}
}

Iată metoda standard. metoda - numele metodei.
public static void vom studia în viitor, dar pentru înțelegere voi explica:

public - modificator care indică faptul că această metodă poate fi folosită oriunde.

static - un modificator care este aplicat câmpului blocului, metodei. Ea indică o legare pe un subiect. În cazul nostru - o metodă.

void - în primul rând, din engleză - acesta este gol, în al doilea rând, void este un tip de date pe care le putem obține ca urmare a metodei. Void este un tip de date, la fel ca int și altele. Acest cuvânt cheie este folosit pentru a returna date mixte, adică simultan String și double și altele. Să creăm o metodă simplă:

public static void method(){
System.out.println("Text dintr-o altă metodă");
}

Și acum vom numi această metodă în metoda principală main

public class Main{
public static void main(String[] args){
method();
}
public static void method(){
System.out.println("Text dintr-o altă metodă");
}
}
--------------------
Text dintr-o altă metodă

Dar se întâmplă că trebuie să transmitem câteva informații metodei. Parametrii pe care îi vom scrie între paranteze ne vor ajuta în acest sens:

public static void method(String name, int age){
System.out.println("My name: " + name);
System.out.println("My age: " + age);
}

Și acum să numim această metodă în metoda principală:

public class Main{
public static void main(String[] args){
method("Ann", 15);
}
public static void method(String name, int age){
System.out.println("My name: " + name);
System.out.println("My age: " + age);
}
}
--------------------
My name: Ann
My age: 15

În metoda principală, metoda metoda a fost apelată și valoarea a fost scrisă acolo. Dar există o nuanță - apelul de metodă trebuie să aibă același număr de argumente și secvență de tipuri ca în descriere, adică am creat o metodă în care au fost înregistrate două tipuri de date - String, int. Când apelăm metoda, trebuie să le specificăm, adică mai întâi specificăm tipul String, apoi int
Să trecem la returnarea unor valori specifice. În loc de cuvântul cheie void, trebuie să specificăm tipul pe care îl dorim, să fie String.

public class Main{
public static void main(String[] args){
String s = "Hi";
s = method(s);
System.out.println(s);
}
static String method(String str){
str=str+"world";
return str;
}
}
--------------------
Hi world

În acest cod, am creat o variabilă s de tip String și am scris textul. În metoda method, am înlocuit argumentele cu one - str de tip String. am atașat lumea textului la variabila str la un șir gol, iar datorită cuvântului cheie return returnăm datele variabilei str. În metoda principală atribuim variabilei s metoda metoda iar între paranteze scriem variabila s pe care o vom scrie în metodă . Pe linia următoare, scoatem variabila s.
