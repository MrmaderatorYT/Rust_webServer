Gestionarea situațiilor de eroare (try-catch)

Sistemul de tratare a erorilor se bazează pe blocul try-catch. Acest bloc va ajuta la procesarea unei anumite părți a codului, care atunci când apare, nu va duce la finalizarea programului, ci pur și simplu va executa codul. Iată un șablon standard pentru dvs.:

try{
cod care poate cauza o eroare
}
catch(classOfError variabilă){
procesarea erorilor
}

Deci acum clarificare. În blocul try, scriem cod care la un moment dat poate să nu funcționeze corect.
În blocul catch, vom prindeți această eroare , specificând ce clasă de eroare va fi detectată, după care descrieți cum să o gestionați.
Clasa unei erori este ceea ce este între paranteze catch
Iată un exemplu:

import java.util.Scanner;
public class Main{
public static void main(String[] args){
Scanner scanner = new Scanner(System.in);
try{
int x = scanner.nextInt();
}
catch(InputMismatchException e){
System.out.println("Nu este un număr");
}

Deci, am creat o variabilă x, datorită căreia vom introduce o valoare de tip int în consolă. Dar există un moment în care introducem un număr greșit și apoi va rula codul din blocul catch. În cazul nostru, textul: Nu este un număr. Iată câteva clase de erori

pentru tine
  • ArithmeticException - eroarea este legată de executarea operaţiilor matematice. Pentru un exemplu simplu, nu putem împărți numerele la zero.
  • ArrayStoreException - eroarea este legată de o încercare de a scrie un obiect de un tip care nu corespunde tipului elementelor matricei în matrice. De exemplu, am creat o matrice de tip String și am scris un element de tipul char.
  • ClassCastException - eroarea este legată de o încercare de a arunca tipuri incorect.
  • IllegalArgumentException - această eroare este asociată cu argumente incorecte ale metodei.
  • NumberFormatException - o eroare care apare atunci când încercați să convertiți o reprezentare textuală incorectă a unui număr într-un număr
  • IndexOutOfBoundsException - o eroare legată, de exemplu, de un index de matrice specificat incorect
  • StringIndexOutOfBoundsException - corespunde unei erori care apare cu un index de element specificat incorect într-un șir de text.
  • NegativeArraySizeException - o eroare legată de o încercare de a crea o matrice de dimensiune negativă
  • NoSuchElementException - o eroare legată de o referire la un element inexistent
  • NullPointerException - eroarea este legată de utilizarea unei referințe goale (valoarea null) în locul unei referințe la obiect< /span >
    • null este folosit pentru a reprezenta absența intenționată a valorilor.
  • UnsupportedOperationException - eroarea este legată de o încercare de a efectua o operație care nu este acceptată.
  • ArrayIndexOutOfBoundsException - apare o eroare la accesarea unui element la un index care depășește intervalul actual de valori.
  • ClassNotFoundException - eroarea este legată de incapacitatea de a accesa clasa.
  • IllegalAccessException - apare o eroare la accesarea resursei.
  • InstantionException - o eroare legată de imposibilitatea creării unui obiect (de exemplu, pe baza unei clase abstracte)
  • InterruptedException - o eroare legată de întreruperea unui fir de către un alt fir.
  • NoSuchFieldException - o eroare legată de absența unui câmp.
  • NoSuchMethodException - eroarea este legată de absența unei metode.
