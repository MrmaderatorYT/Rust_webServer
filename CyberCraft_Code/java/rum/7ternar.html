Operator ternar

Operator ternar - înlocuirea blocului if-else. Adică, această declarație realizează munca blocului if și a blocului else, adică a blocului else-if block, operatorul ternar nu acceptă

Iată vizualizarea standard a operatorului ternar condiționat:

expression_#1 ? expression_#2 : expression_#3;

Acum hai să ne dăm seama

Expresia #1 - denotă orice expresie, al cărei calcul dă o valoare logică (tip boolean), dar pentru starters vom crea o variabilă a, iar în expresie vom crea o altă variabilă:

int a = 10;
int b = ? expression_#2 : expression_#3;

Am creat o variabilă b de tipul int. După equals, creăm o verificare pentru expresie, care este aceeași cu condiția din paranteze. O afecțiune este de obicei indicată în paranteze, dar nu neapărat:

int a = 10;
int b = (a>5) ? expression_#2 : expression_#3;

În expresia expresia #2 și expresia #3 - execută codul.
Expresia #2 este codul care este executat atunci când condiția adevărată este adevărată, adică codul care se află în if bloc.
Expresia #3 este codul care se află în blocul else, adică codul care va fi executat dacă expresia #2 va primi o eroare.

Deci, aici este codul complet folosind operatorul ternar:

int a = 10;
int b = (a>5) ? ++a : --a;

Aici punem punct și virgulă la sfârșitul expresiei.
Există o nuanță când se utilizează operatorul ternar: acest operator funcționează numai cu alocarea unei variabile noi de același tip sau a aceleiași variabile:

int a = 10;
a = a>5 ? ++a : --a;

Si intre:

int a = 10;
int b = (a>5) ? ++a : --a;

Nu va fi nicio diferență.
De asemenea, există și între program: ce muncă a fost făcută? Un operator ternar sau un operator condițional.... Viteza de execuție este aceeași.
