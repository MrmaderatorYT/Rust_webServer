Referință pentru metodă și clasă

Dacă vorbim despre executarea unei referințe la o metodă non-statică a unui obiect specific și numele metodei este indicat prin operatorul ::. Adică, modelul de execuție al unei referințe la o metodă non-statică este următorul:

obiect :: nume obiect;

Aceasta a fost referința la metoda obiectului, acum să trecem la referința la metoda non-statică a clasei. În acest din urmă caz, contează dacă metoda este statică sau nu. O referire la o metodă nestatică dintr-o anumită clasă este indicată în următoarea formă:

Class :: nameMethod;

Referință pentru metoda statică...
O referire la o metodă statică este în mod formal aceeași cu o referire la o metodă non-statică a unei clase, dar consecințele sunt diferite. Referințele la metode pot fi scrise ca o expresie lambda. De exemplu, o referire la metoda unui obiect care utilizează o expresie lambda arată astfel:

argumente -> object.method(argumente)

Dacă scrieți o referință la o metodă de clasă non-statică ca expresie lambda, arată astfel:

(obiect, argumente) -> object.method(argumente)

Și o referire la o metodă statică printr-o expresie lambda arată astfel:

(arguments) -> class.method(arguments)

De exemplu, referință la o metodă statică:

System.out::println

Echivalent cu:

t -> System.out.println(t);

Iată un exemplu de referință la o metodă non-statică a unei clase:

MyClass :: method

Și aceasta este echivalentă cu:

(MyClass obj, int num, char symb)->
obj.method(num, symb)

Să trecem la ultimul link: către constructor. Arată astfel:

class::new

După cum puteți vedea, clasa este scrisă cu litere mici, așa că acesta este numele clasei. Această referință este echivalentă cu următoarea expresie lambda:

(argumente) -> new clasă (argumente)
