Modificator transient

Acest modificator este utilizat în procesul de serializare și deserializare a obiectelor. Să presupunem că avem un obiect care are variabile, iar aceste variabile sunt valori. Toate acestea se numesc starea obiectului.
Serializare - conversia stării obiectului într-o secvență de octeți. Acești octeți sunt de obicei stocați într-un fișier.
Deserializarea este procesul invers al serializării.
transient este util în mai multe scenarii:
1. Când îl utilizați pentru variabile derivate.
2. Acest lucru este util pentru variabilele care nu reprezintă starea unui obiect
3. Pentru orice referințe neserializate.
Pentru a efectua serializarea, trebuie să implementați interfața Serializabilă:

class Class implemente Serializabile{
//code
}

Scopul acestui modificator este atunci când este nevoie de a salva un obiect de clasă. Modificatorul transient este setat înainte de declararea unei variabile de clasă internă, a cărei valoare nu trebuie păstrată în cazul salvării obiectului. Datorită acestui modificator, puteți specifica date care nu trebuie să fie stocate la serializarea obiectului:

class Book{
String name;
String author;
int year;
transient String ID;
}

În acest caz, variabila id nu este salvată pentru scriere în fișier
