Valoare null

null este o valoare, nu un tip, deci, de exemplu, în acest cod, variabila a are valoarea 5:

int a = 5;

La fel ca și 5, null are valoare. De asemenea, null este folosit doar pentru obiecte, adică primitive precum int și așa mai departe, nu vom putea folosi null:

char a = null; //eroare

Adică, această valoare este utilizată în felul următor:

String a = null;

De asemenea, fiecare primitivă are o valoare implicită, de exemplu int are numărul 0 și boolean are false, atunci valoarea implicită a tuturor obiectelor este null. De asemenea, trebuie remarcat faptul că null nu este un obiect și nu este din nou un tip. Este doar o valoare specială pentru obiecte. Putem folosi chiar această valoare pentru orice tip, de exemplu:

String a = null;
Integer b = null;
String a1 = (String)null;
Integer b1 = (Integer)null;

Așa cum am spus, null nu poate fi folosit pentru tipurile primitive, deoarece valoarea null nu poate fi turnată la acea valoare , pe care le-am specificat în tip, adică:

double a = null; //eroare

Dar fiecare primitiv are o clasă wrapper. Și dacă creăm un obiect bazat pe o clasă shell și apoi atribuim unui tip primitiv, obținem o eroare de compilare NullPointerException:

Integer a = null;
int b = (int)a;

Totul se datorează ambalării automate.

Ambalare automată, despachetare

Autopacking - conversia tipurilor primitive în obiecte echivalente, implementată automat.
Despachetarea este procesul invers al împachetarii automate.

Adică toate acestea pot fi specificate ca: procesul de transformare a obiectelor în tipurile lor primitive corespunzătoare. Un exemplu de cod pentru împachetare și despachetare automată este prezentat mai jos:

Integer integer = 9; // autopack

int in = 0;
in = new Integer(9); // despachetați

Când se utilizează împachetarea și despachetarea automată?

Autopacking este aplicat de compilatorul Java în următoarele condiții:

  • Când o valoare de tip primitiv este transmisă unei metode ca parametru de metodă care așteaptă un obiect din clasa wrapper corespunzătoare.
  • Când o valoare de tip primitiv este atribuită unei variabile din clasa shell corespunzătoare.

Să revenim la operatorul instanceof. Va returna false în toate cazurile dacă variabila obiect este null. De asemenea, putem compara valorile null folosind operatorii == și !=, dar nu cu operatorii aritmetici și logici.
Dacă comparăm două valori null între ele, vom obține valoarea true.
