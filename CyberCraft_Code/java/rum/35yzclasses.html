Clasuri și metode generice

O Class generalizată este descrisă ca o Class normală, dar, desigur, există anumite particularități. În descrierea clasei, adăugăm un anumit parametru formal (sau parametri) și sunt declarați ca fiind cel care denotă tipul de date, iar în cazul creării unui obiect al clasei, identificatorul de tip este trecut ca parametru. . Parametrii care sunt identificați în descrierea clasei generalizate cu tipul (le voi numi parametri generalizați) sunt indicați între paranteze unghiulare în descrierea clasei după numele clasei:

class Nume {
//descriere
}

Clasele generalizate sunt folosite în astfel de cazuri, când o anumită „structură” sau „construcție” este implementată cu ajutorul unei clase, care are caracteristici universale și este slab dependentă de tipul de date de tratat. Parametrii de tip sunt pur și simplu enumerați cu virgule și acești parametri sunt utilizați în descrierea generică a clasei (ca identificatori de tip). Ca parametri de tip, în principiu, poate fi folosit orice identificator - principalul lucru este că nu este unul dintre cuvintele cheie înregistrate (cum ar fi int și altele):

class Class <X, Q>{
//descriere
}

La crearea unui obiect bazat pe o Class generică, parantezele unghiulare din comanda de creare a variabilei obiect indică identificatorii de tip care ar trebui utilizați în locul parametrilor generici. De asemenea, identificatorii sunt specificați între paranteze unghiulare în instrucțiunile de creare a obiectului - între numele clasei și parantezele rotunde cu argumentele constructorului:

class Class <X, Q>{
Class <X, Q> obj = new Class <X, Q> (arguments);
}

Există și o versiune simplificată:

class Class <X, Q>{
Class <X, Q> obj = nou Class < > (arguments);
}

În acest caz, înlocuirea parametrilor generalizați în cazul creării obiectului este determinată pe baza identificatorilor specificați în primele paranteze unghiulare din primele paranteze unghiulare (în instrucțiunile de declarare a variabilei obiect).

Moștenirea claselor generalizate

Pentru a imita clasele universale, este necesar ca clasa principală și superclasa să aibă aceiași parametri:

class Class <X, Q>{
int a;

clasa Main <X, Q> extends Class <X, Q>{
//code
}

De asemenea, clasa principală poate adăuga și utiliza unii dintre parametrii proprii de tip:

clasa Class <X, Q>{
int a;

clasa Main <X, Q, Z> extends/span> Class <X, Q>{
//code
}

Este posibil să existe și următoarea situație: când o Class poate să nu fie generalizată deloc:

class Class <X, Q>{
int a;

class Main extends Clasa <Integer>{
//code
}

Metode generalizate

De obicei, metodele generice sunt statice. Sunt create în același mod ca și cele obișnuite, dar cu parametri.

static typeResult name (argumente){
//code
}

Iată cea mai simplă vizualizare pentru dvs.:

static <X> void show(){
//code
}
