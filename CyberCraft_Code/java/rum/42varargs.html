Varargs

Varargs - argumente de lungime variabilă. Ele ne permit să creăm metode cu un număr arbitrar de argumente. Să creăm un exemplu. Să presupunem că trebuie să scriem o metodă care va primi un număr arbitrar de argumente multiple și să le adunăm împreună. Avem 2 opțiuni:
1. Supraîncărcare:
class Calc{
int sum (int a, int b){/*code*/}
int sum (int a, int b, int c){/*code*/}
int sum (int a, int b, int c, int d){/*code*/}
}

Supraîncărcare - metode care au același nume, dar de tipuri diferite și/sau cu un număr diferit de parametri. Dar folosind supraîncărcare, avem 2 probleme. În primul rând - uneori nu este clar când să se oprească, iar în al doilea rând - mult.

Există mai multe reguli pentru utilizarea varargs:
1. Varargs pot fi specificate numai pentru o anumită metodă:

void a(String... args){
//code
}
2. Fiecare astfel de argument din corpul metodei este o matrice, așa că puteți lucra la el ca pe o matrice obișnuită.
3. Argumentul varargs trebuie să fie ultimul din lista de argumente:

void a(String... a, String b) //acest lucru nu este posibil

void a(String a, String... b) //yes you can

Împreună

Voi pune din nou întrebarea: când să folosesc varargs? Răspunsul depinde de unde programează persoana și cine întreabă. Dacă o astfel de întrebare este pusă de un client al unui API care are metode varargs, atunci folosiți astfel de metode cât mai des posibil. Dacă dezvoltatorul API pune această întrebare, atunci ar trebui să le utilizați atunci când beneficiul este evident, adică nu ar trebui să le folosiți des. De asemenea, nu se recomandă utilizarea supraîncărcării. metode, deoarece clienții vor avea dificultăți în a înțelege care dintre metodele supraîncărcate este utilizată.
