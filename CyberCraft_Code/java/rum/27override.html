Metode de suprascrie

Se întâmplă că într-o clasă există multe metode cu același nume, dar cu argumente diferite. Să ne imaginăm că avem o metodă care ar trebui să facă lucruri diferite în funcție de clasa în care a fost executată:

public class Car{
public void Car1(){
System.out.println("Car");
}
}

Aceasta este clasa părinte și vom crea altele în fișiere separate: One, Two

public class One extends Car{
@Override
public void car1(){
System.out.println("primul");
}
}

Da clasa Two:

public class Two extends Car{
@Override
public void car1(){
System.out.println("prieten");
}
}

Și acum, în metoda principală, clasa Main, vom apela clase prin crearea unui obiect:

public class Main{
public static void main(String[] args){
Car car1 = new One();
Car car2 = new Two();
car1.car1();
car2.car1();
}
}

primul
prieten

Să trecem la nuanțe:

  • Tipul metodei suprascrise nu poate fi diferit de cel original
  • Modificatorul de acces trebuie să fie, de asemenea, același
  • Metoda suprascrisă trebuie să aibă aceleași argumente ca metoda principală
