Să creăm o clasă Children:

public class Main{
public static void main(String[] args){
}
}
class Children{
int age;
String name;
void print_info(){
System.out.println("Nume" + name);
}
}

Aici am creat o metodă, dar metoda nu este publică, ceea ce înseamnă că nu poate fi folosită în alte clase.
Constructorii sunt o alternativă la metodele obișnuite. Acestea sunt metode atât de speciale. Ele sunt apelate atunci când este creat un nou obiect din această clasă. Constructorii inițializează obiectul și poartă numele clasei. Avem o clasă Children și vom elimina metoda și vom scrie constructorul:

public class Main{
public static void main(String[] args){
}
}
class Children{
int age;
String name;
public Children(String name, int age){
this.name = name;
this.age = age;
}
}

this este un cuvânt cheie care specifică o referință la instanța curentă a clasei. Prin acest cuvânt cheie, putem accesa variabilele și metodele obiectului, precum și apelăm constructorilor acestuia. După cuvântul cheie, punem un punct și ne referim la variabila pe care am creat-o și i-am atribuit celei pe care am creat-o în constructor. Să mergem la clasa principală Main. Să trecem la clasa Children. Pentru a fi sincer, creăm un obiect bazat pe clasa Children, așa că scriem:

public class Main{
public static void main(String[] args){
Children children = new Children(10, "Ann");
}
}
class Children{
int age;
String name;
public Children(String name, int age){
this.name = name;
this.age = age;
}
}

Dar în paranteze, trebuie să scrieți datele, și anume vârsta, adică numele - nume. Este necesar să scrieți în ordinea așa cum am scris în metodă. Apoi ieșim în consolă:

public class Main{
public static void main(String[] args){
Children children = new Children(10, "Ann");
System.out.println(children.age);
System.out.println(children.name);
}
}
class Children{
int age;
String name;
public Children(String name, int age){
this.name = name;
this.age = age;
}
}

Să trecem la încapsulare.
Încapsulare - ascunderea datelor în mijlocul unui anumit domeniu. Pentru aceasta sunt folosite metode de acces. Aceștia se numesc Getters și Setters(Getters și Setters). Am creat o variabilă int în clasa Children. Dacă scriem modificatorul de acces private în fața acestei variabile, nu vom putea scoate variabila în clasa Main . Și în general... Nu îl vom putea folosi la alte clase. Pentru asta sunt Getter și Setter. Vom înlocui clasa Children:

public class Main{
public static void main(String[] args){
Children children = new Children("Ann");
children.setAge(10);
System.out.println(children.getAge);
System.out.println(children.name);
}
}
class Children{
public int getAge() {
return age;
}
public void setAge(int age) {
this.age = age;
}
public String getName() {
return name;
}
public void setName(String name) {
this.name = name;
}
private int age;
String name;
public Children(String name){
this.name = name;
}
}
--------------------
10
Ann

Am creat getter-ul și setter-ul în clasa Children. Am creat pentru variabilele vârsta și nume. De ce am declarat o variabilă private unei variabile și am folosit constructori când putem declara o variabilă public tuturor?
Tocmai sa întâmplat... Chiar și foarte des, în timpul execuției programului, o valoare incorectă va fi transmisă variabilei. De exemplu, o valoare negativă va fi transmisă pentru valoarea de vârstă, care nu ar trebui să fie.
