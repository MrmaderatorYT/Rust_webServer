Interfețe

Interfața este o copie trunchiată a unei clase abstracte. Interfața conține declarația de metode și/sau constrângeri statice. Metodele sunt declarate doar în interfață. Procesul de implementare a unei interfețe într-o clasă seamănă cu moștenirea unei superclase abstracte într-o subclasă. In cazul interfetelor, clasa care implementeaza interfata trebuie sa contina o descriere a tuturor metodelor declarate in interfata. O circumstanță importantă este legată de faptul că aceeași clasă poate implementa mai multe interfețe în același timp. Implementarea interfețelor este descrisă în același mod ca o clasă normală, dar în loc de cuvântul cheie class, trebuie să scriem interface .
De ce interfețe?
Faptul este că moștenirea multiplă este interzisă în Java: o subclasă poate avea una și o singură superclasă. Aceasta este o limitare serioasă, deoarece, de exemplu, în limbajul C++, o clasă poate moșteni mai multe clase în același timp.
Un alt punct cheie: toate metodele declarate în interfață sunt publice. Adică, modificatorul de acces public nu este necesar.

interface MyInterface{
int number = 100;
}

Să creăm un exemplu simplu de implementare a interfețelor:

interface A{
int number = 50;
int getNumber(double x);
char getSymbol(int y);
}
class MyClass implements A{
public int getNumber(double x){
return (int) x;
}
public char getSymbol (int n){
return (char)('A' + n);
}
}
class Main{
public static void main(String[] args){
MyClass obj = new MyClass();
System.out.println("Constanta statica: " + MyClass.number);
System.out.println("Întreg: "+obj.getNumber(20.5));
System.out.println("Simbol: "+obj.getSymbol(3));
}
}
-----------
Constanta statica: 50
Număr întreg: 20
Simbol: D

return(int) x - tipuri de difuzare: variabilă x de tip double și l-am înlocuit cu int
return (car)('A' + n); - am tradus numărul de serie al scrisorii în text.
