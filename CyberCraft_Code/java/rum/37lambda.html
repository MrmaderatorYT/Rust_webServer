Expresii lambda

Expresii Lambda - un anumit bloc de cod care definește o acțiune. De asemenea, puteți considera expresiile lambda drept definiția unei metode anonime (lecția următoare), adică o metodă care nu are un nume. De asemenea, nu au niciun tip de rezultat. În general, expresiile lambda sunt un bun contor pentru o clasă anonimă, așa că este mai bine să folosiți expresii lambda. Un bloc de comenzi dintr-o expresie lambda se numește corpul unei expresii lambda. În consecință, expresia lambda constă din argumente corporale. Această expresie este construită în același mod ca o expresie regulată, dar fără numele și identificatorul tipului de rezultat. Comenzile din corpul unei expresii lambda sunt descrise de comenzi (da, logica #1). Argumentelor li se dă un tip (deși în unele cazuri nu). Comenzile sunt scrise între paranteze și o „săgeată” este indicată între argumente și blocul de comandă: ->. Și pe baza acestui lucru, putem obține următoarele:

(argumente) -> {
echipe
}
Și acum să trecem la vizualizarea standard, să spunem asta:
(int x, int y) -> {
int z = x+y;
System.out.println(z);
}

Există câteva reguli care vă permit să simplificați sintaxa descrierii expresiilor lambda:

  • Tipul de argument poate fi omis dacă poate fi identificat înaintea expresiei lambda
  • Dacă expresia lambda are un argument, al cărui tip nu este specificat, atunci parantezele nu pot fi folosite
  • Dacă expresia lambda nu are argumente, se folosesc paranteze goale.
  • Dacă există o singură comandă în corpul expresiei lambda, atunci bretele pot fi omise. În acest caz, corpul expresiei lambda constă dintr-o instrucțiune return, dar acest cuvânt cheie poate fi omis

Interfețe funcționale

Interfețele funcționale sunt interfețe cu o singură metodă abstractă. Adică nu există nimic special. Trebuie îndeplinită o singură condiție: o metodă abstractă este declarată în interfață. Particularitatea interfeței funcționale în ceea ce privește utilizarea expresiilor lambda se datorează faptului că expresiei lambda i se poate atribui o valoare unei variabile de tipul interfeței - dar cu condiția ca interfața să fie funcțională și semnătura metodei abstracte. în interfața funcțională corespunde parametrilor expresiei lambda - este creat de un obiect bazat pe o clasă anonimă care implementează această interfață, iar codul expresiei lambda este folosit ca cod pentru a defini o metodă abstractă
