Operator condiționat

Operator condiționat sau structură if-else

Operatorul condiționat vă permite să executați diferite blocuri de comenzi în funcție de o anumită condiție, adevărată sau falsă.

Pentru a declara un operator condiționat, trebuie să scrieți cuvântul cheie if Iată vizualizarea standard a operatorului condiționat:

if (condiție){
cod
}

Сhai sa facem un exemplu:

import java.util.Scanner;
public class Main{
public static void main(String[] args){
Scanner scanner = new Scanner(System.in);
int a = scanner.nextInt();
if (a>100){
System.out.println("Число більше 100");
}
}
}

Aceasta este o formă simplificată a declarației condiționate. Daca introducem un numar mai mare de 100, intram in consola: Numar mai mare de 100, dar daca introducem un numar mai mic sau egal cu 100, nu mai obtinem nimic datorita structurii simplificate.
O clauză completă are un cuvânt cheie else
Structura completă a declarației condiționale arată astfel:


if (condiție){
cod
}
else {
cod
}

Deci, logica de execuție a instrucțiunii condiționate este următoarea: dacă condiția este în bloc if adevărat, codul din bloc este executat if, dacă condiția din paranteze nu este adevărată, adică falsă, blocul este executat else in acest bloc nu punem conditii, deoarece se va executa cand blocul nu este executat if.
Să creăm un exemplu:

int a=5;
if (a<4){
System.out.println("no")
}
else {
System.out.println("yes")
}

Aici, am creat o variabilă numerică a și i-am atribuit numărul 5. Pe rândul următor, am creat o verificare: dacă variabila a este mai mică de 4. Dacă da, atunci vom primi textul nu în consolă, dacă nu (mai mare sau egal cu 4), vom primi da.

< br>

Să trecem la un alt tip de operatori - cei imbricați. Un tip imbricat are un cuvânt cheie suplimentar else if

if (condiție){
cod
}
else if(condiție){
cod
}
else {
cod
}

else if este scris după blocul if. Pot exista multe blocuri else if, chiar și 100. Logica de execuție este următoarea: dacă condiția din blocul if este falsă, else if, dacă în acest bloc este executată o condiție falsă, se execută un alt bloc și așa mai departe, dacă există un else blocați, apoi după toate condițiile false else if codul din blocul else este executat, dacă nu există else bloc, nu vom primi nimic.

int a = 11;
if (a<11){
System.out.println("first");
}
else if(a==10){
System.out.println("second");
}
else if(a==12) {
System.out.println("third");
}

Aici am creat o variabilă și am atribuit numărul 11. Apoi am creat prima verificare dacă variabila este mai mică de 11, atunci vom primi mai întâi textul în consolă dacă obținem o valoare falsă, adică dacă variabila a este mai mare sau egal cu 11. Următorul bloc este dacă variabila este 10, iar al treilea dacă este 12. În acest cod, nu obține nimic, deoarece verificarea a trei blocurile nu se află în intervalul numărului 11, ci al blocului else niciunul

int a = 0;
if (a==0){
System.out.println("first");
}
else if(a>=0){
System.out.println("second");
}
else{
System.out.println("third");
}

După cum puteți vedea, avem două verificări. Vom primi primul și totul în consolă, adică numai acesta, nu primul și al doilea, deoarece variabila noastră este inclusă și în a doua verificare. Nu vom primi o valoare, deoarece după primirea valorii true adică adevărul, verificarea este finalizată

.

Există un alt exemplu:

int a = 0;
if (a==0){
System.out.println("first");
}
if(a>=0){
System.out.println("second");
}
else{
System.out.println("third");
}

Aici vom primi două valori în consolă, și anume prima și a doua, deoarece există două blocuri if. Primul bloc este vizualizarea simplificată, iar al doilea bloc if este cel complet, adică else este atașat ultimului if.
