Pipe

Pipe - оператор дозволяє ланцюговано виконувати послідовні операції над даними, роблячи код більш читабельним та ефективним. Тобто таким чином ми будуємо ланцюг послідовних дій. Зазвичай, він використовується в бібліотеці dplyr. В цьому пакеті є функція:

my_summarise <- function(data, group_var){
data %>%
group_by({{group_var}})%>%
summrise(mean = mean(mass))
}

У функції my_summarise використовуються дві основні функції для маніпуляції даними: group_by та summarise. Оператор %>% (pipe) слугує для ланцюгового виконання цих функцій.

. Розберемо код:
group_by({{ group_var }})
Ця частина групує дані за змінною, яку ви передаєте в функцію як group_var. Ви використовуєте {{ group_var }}для того, щоб використовувати змінну group_var внутрішньо в group_by.
summarise(mean = mean(mass))
Ця частина обчислює середнє значення змінної mass в кожній групі, створеній функцією group_by. Результатом є новий стовпець mean, який містить середнє значення mass для кожної групи.

А тепер, з оператором pipe

Оператор %>% (pipe) ланцюгово передає результати з однієї функції до іншої. У нашому випадку: Вхідні дані data передаються до group_by. Результат від group_by передається до summarise. Таким чином, функція my_summarise приймає датафрейм data і назву змінної group_var, групує дані за цією змінною і обчислює середнє значення змінної mass для кожної групи.
