Git Commit

Оскільки ми завершили нашу роботу, ми готові перейти з stage до commit для нашого репо. Додавання збережень відстежує наш прогрес і зміни під час роботи. Git розглядає кожну commit точку зміни або «точку збереження». Це частина проекту, до якої ви можете повернутися, якщо виявите помилку або захочете внести зміни.
Коли ми юзаємо commit, ми завжди повинні включати повідомлення про цей коміт.
Додаючи чіткі повідомлення до кожного коміту, вам (та іншим) буде легко побачити, що змінилося і коли.

git commit -m "First release of Hello World!"
[master (root-commit) 221ec6e] First release of Hello World!
3 files changed, 26 insertions(+)
create mode 100644 README.md
create mode 100644 bluestyle.css
create mode 100644 index.html
Команда commit виконує зберігання та додає повідомлення. завдяки прапору -m і потім можемо впихнути текст: "message"
Середовище Staging Environment було додано до нашого репо з повідомленням:
"First release of Hello World!"
