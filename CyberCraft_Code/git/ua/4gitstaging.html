Git staging

Однією з основних функцій Git є концепції проміжного середовища та коміту.

Під час роботи ви можете додавати, редагувати та видаляти файли. Але щоразу, коли ви досягаєте певної віхи або завершуєте частину роботи, ви повинні додати файли до проміжного середовища.

Поетапні файли — це файли, готові до розміщення в репозиторії, над яким ви працюєте.

git add data.txt
Наразі ми закінчили працювати з data.txt. Таким чином, ми можемо додати його до середовища Staging. Файл має бути Staged . Перевіримо статус:
git status
On branch master

No commits yet

Changes to be committed:
(use "git rm --cached ..." to unstage)
new file: data.txt
Тепер файл додано до проміжного середовища.

Додавання більше файлів (не один, а більше)

Ви також можете створити кілька файлів одночасно. Давайте додамо ще 2 файли в нашу робочу папку. Знову скористайтеся текстовим редактором.
Файл README.md, що описує репозиторій (рекомендовано для всіх репозиторіїв (відкритих)). Умовно у нас є такий README.md
#data.txt
this file is used for ML
І тепер умовно у нас є файл data.dat. Ці файли в одній теці. Тепер робимо це:
git add --all
-all Замість окремих імен файлів буде використано stage всі зміни (нові, змінені та видалені) файли.
git status
On branch master

No commits yet

Changes to be committed:
(use "git rm --cached ..." to unstage)
new file: README.md
new file: data.dat
new file: data.txt
Тепер усі 3 файли додано до проміжного середовища, і ми готові створити наш перший файл commit. І цей во: git add --all можна записати по іншому: git add -A
